# Rent Payment Checker Agent

Automated monthly rent payment verification across rental properties.
Scrapes transaction data from Monarch Money, applies a three-step matching
pipeline, and emails a status summary.

## How It Works

1. **Idempotency check** — skips if this month was already checked
2. **Monarch scraping** — extracts current-month transactions via Playwright
3. **Three-step matching** per property:
   - Step 1: Exact Monarch category label match (deterministic)
   - Step 2: Amount fallback on all transactions (deterministic)
   - Step 3: Ollama LLM review for unresolved properties (optional)
4. **Email notification** — body generated by LLM (or plain-text fallback)
5. **Run log** — appended to `logs/run_history.json`

---

## Prerequisites

- Python 3.13
- [Playwright](https://playwright.dev/python/) — `pip install playwright && playwright install chromium`
- [Ollama](https://ollama.ai/) running locally with `qwen3:8b` pulled
- A Gmail account with an [App Password](https://myaccount.google.com/apppasswords) enabled

## Setup

### 1. Clone and install dependencies

```bash
git clone <repo-url>
cd personal-finance-rent-checker-agent
pip install playwright
playwright install chromium
```

### 2. Create config files

```bash
# Business rules
cp config/agent_config.json.example config/agent_config.json

# Secrets
cp config/.env.json.example config/.env.json
```

Edit `config/agent_config.json` with your actual rent amounts, due dates,
tenant names, and Monarch category labels for each property.

Edit `config/.env.json` with your Gmail credentials.

### 3. Set up the Playwright browser profile (one-time)

The scraper uses a persistent browser profile so you only need to log in to
Monarch Money once:

```bash
python - <<'EOF'
from playwright.sync_api import sync_playwright
import json, pathlib

cfg_path = pathlib.Path("config/agent_config.json")
profile = json.loads(cfg_path.read_text())["scraper"]["browser_profile_path"]
print(f"Creating profile at: {profile}")

with sync_playwright() as p:
    ctx = p.chromium.launch_persistent_context(profile, headless=False)
    page = ctx.new_page()
    page.goto("https://app.monarchmoney.com/login")
    input("Log in to Monarch Money, then press Enter to save the session...")
    ctx.close()
print("Profile saved.")
EOF
```

### 4. Verify configuration

```bash
python -m src.config_loader
```

This validates all three config sources and prints a summary without running
the scraper or sending any email.

## Running

### Normal run

```bash
python -m src.orchestrator
```

### Common flags

| Flag | Description |
|---|---|
| `--dry-run` | Run everything but print email to stdout; do not send or log |
| `--force` | Re-run even if this month is already complete |
| `--verbose` | Set log level to DEBUG |
| `--no-headless` | Show the Chromium window during scraping |
| `--no-scrape` | Skip scraping (use with `--transactions-file` for testing) |
| `--transactions-file FILE` | JSON fixture file to use with `--no-scrape` |

### Examples

```bash
# Preview what would happen without sending email
python -m src.orchestrator --dry-run

# Debug scraper with visible browser
python -m src.orchestrator --no-headless --dry-run

# Test email pipeline with fixture data
python -m src.orchestrator --no-scrape --transactions-file tests/fixtures/sample.json --dry-run

# Force re-run this month
python -m src.orchestrator --force
```

## Scheduling (Windows Task Scheduler)

To run automatically on the 5th of each month:

1. Open Task Scheduler → Create Basic Task
2. Trigger: Monthly, on day 5
3. Action: Start a program
   - Program: `C:\path\to\python.exe`
   - Arguments: `-m src.orchestrator`
   - Start in: `C:\path\to\personal-finance-rent-checker-agent`

The idempotency check prevents double-runs if the task fires more than once.

## Troubleshooting

### "Monarch redirected to login page"
The browser session has expired. Run the profile setup command again
(step 3 above) to re-authenticate.

### "Scraper found no transactions"
Monarch's HTML structure may have changed. Check `logs/scraper_*.html` for
a saved snapshot of the page and update the selectors in
`src/monarch_scraper.py`. Document the fix in `LESSONS.md`.

### "Ollama unavailable"
Start Ollama: `ollama serve`. The agent will fall back to a plain-text
email template and continue without LLM features.

### "Config error: missing or invalid field: ..."
Check the named field in `config/agent_config.json` or `config/.env.json`.
Run `python -m src.config_loader` for a full config diagnostic.
